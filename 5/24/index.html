ビルド(実践)
前回はソースコードをビルドする手順を教わったよ！
今回は実際にソースコードをビルドしよう！
必要パッケージインストール
今回は GNU Hello をビルドしよう！ 「Hello, world!」と表示するコードをパッケージ化したものだよ
初心者向けでちょうど良い題材だね！
まずは必要となるパッケージ gcc と make をインストールしよう！ 
sudo apt install -y gcc make
OK！ インストールできたよ！
ダウンロード
次に GNU Hello のソースコードをダウンロードしよう 作業用ディレクトリを作りそこへ移動しておいてね 
mkdir -p ~/hello
cd ~/hello
ディレクトリを作って移動したよ！
そうしたら 公式ページ からソースコードを取得しよう バージョン番号は変数に分けておくと便利だよ 
VERSION=2.12.1
curl -LO "https://ftp.gnu.org/gnu/hello/hello-$VERSION.tar.gz"
ダウンロードが終わったよ 拡張子が .tar.gz ということは tarball なんだね！
そのとおり！ そのままでは使えないので tar で展開して 展開されたディレクトリへ移動しよう！ 
tar xf "hello-$VERSION.tar.gz"
cd "hello-$VERSION"
いよいよソースコードとご対面だね
あっ！ README ファイルもある！
README を読んでビルドに必要な情報を確認しておこう！ 
less README
configure
次は configure スクリプトだよ まずはどんなオプションがあるか確認しよう 
./configure --help
たくさんオプションがあるけどどれを使えばいいの？
一般的にはインストール先を指定する --prefix オプションを使うよ 以下のようにして実行しよう 
./configure --prefix=/usr
うん！ /usr へインストールするようにしたよ！
make
次に make を実行しよう 
make
おお！ hello という実行ファイルができた！ これはもう動くの？
実行ファイルは完成しているよ 以下のように動かしてみよう 
./hello
うん！ やってみるね！ 
Hello, world!
実行ファイルが期待通りに動いているね！ ビルド成功だよ！
make install
最後に make install を実行しよう /usr 配下にファイルを配置するのでsudo が必要になるよ 
sudo make install
これでシステム全体で hello が使えるようになったんだね
実行ファイルと一緒に関連ファイルも適切な場所に配置されるんだ 例えば hello のマニュアルも見れるようになっているよ 
man hello
まとめ
今回は GNU Hello をビルドしてインストールしたよ！
パッケージ管理システムから手に入るパッケージは 大多数のユーザが使えるよう無難な設定のビルドで作られているんだ ソースコードからビルドすることで特別な機能を有効にしたり 自分の環境に最適化した実行ファイルを作ることができるよ！
