# アーキテクチャ

しげる（通常）：
  今回は CPU に大きく関わる
  「アーキテクチャ(archtecture)」を知ろう！
みどり（通常）：
  `arch` で出力されるものだよね

```bash
arch
```

みどり（通常）：
  実行結果はこうだったよ！

```console
x86_64
```

みどり（通常）：
  この出力にはどんな
  意味があるんだろう？

## 種類

しげる（通常）：
  「アーキテクチャ」は建築様式や構造を意味するんだ
  Linux の場合は CPU の設計や構造の名前だと
  思ってもらえればいいよ
みどり（通常）：
  アーキテクチャの種類には
  どういうものがあるのかな？
しげる（通常）：
  代表的なアーキテクチャを紹介するね

アーキテクチャ     | ビット数 | デザイン | 設計方針                   | 主な用途
------------------ | -------- | -------- | -------------------------- | --------
`x86`              | 32 bit   | CISC     | 消費電力や発熱より性能優先 | PC 向け CPU
`x86_64`, `x64`    | 64 bit   | CISC     | 同上                       | 同上
`aarch`, `arm`     | 32 bit   | RISC     | 消費電力あたりの効率優先   | スマートフォンや組み込み向け CPU
`aarch64`, `arm64` | 64 bit   | RISC     | 同上                       | 同上

みどり（通常）：
  ビット数って何かな？
  現在は 64 bit が主流と聞くけど…
しげる（通常）：
  ビット数は CPU が処理をする信号の単位を表すよ
  ビット(bit) は 0 または 1 の二値が表せる情報量の単位だね
しげる（通常）：
  例えばビット数 32 bit なら
  二進数 32 桁を一つの単位として処理するんだ
  ビット数が大きいとそれだけ大きな数値を表現できるので
  CPU が扱える数値や認識できるデータサイズなどに影響するよ
  ただビット数は処理速度には直接影響ないことは注意しよう
みどり（通常）：
  RISC や CISC って？
しげる（通常）：
  CPU が処理できる命令をどのように
  持たせるかの方向性のことなんだ
しげる（通常）：
  「CISC(complex instruction set computer)」は
  CPU に多数の複雑な命令を持たせるんだ
  多くの処理が一つの命令でこなせる反面
  CPU の規模が大きくなり高価になる傾向があるよ
しげる（通常）：
  「RISC(reduced instruction set computer)」は
  CPU に少数の単純な命令を持たせるんだ
  それらを組み合わせて複雑な処理を実現するよ
しげる（通常）：
  ただし、よほど CPU の深くに立ち入らない限り
  RISC と CISC は概念だけ知っていれば十分だよ

## まとめ

みどり（通常）：
  今回はアーキテクチャを学んだよ
  CPU 設計の呼び名なんだ！
しげる（通常）：
  みどりが言っていたとおり最近は 64 bit が主流だよ
  まずは 64 bit アーキテクチャの
  `x86_64` と `aarch64` を覚えておこう！
