# `grep` コマンド

みどり（通常）：
  お兄ちゃん！
  今回は正規表現を使ったコマンドを
  教えてくれるんだよね？
しげる（通常）：
  そうだよ！
  今回は `grep` を学ぼう！

## 使い方

しげる（通常）：
  `grep` はファイルの内容から
  指定された正規表現を検索して
  マッチした行を表示してくれるよ！

```bash
grep [オプション]... 正規表現 [ファイル]
```

オプション                    | 役割
----------------------------- | ----
`-E`, `--extended-regexp`     | 拡張正規表現を使って検索(`+` や `?` などが使える)
`-F`, `--fixed-strings`       | 正規表現を使わず指定された文字列で検索
`-o`, `--only-matching`       | 「マッチした行」の代わりに「正規表現にマッチした箇所」を表示
`-m 回数`, `--max-count=回数` | 指定された回数見つかった時点で終了
`-q`, `--quiet`, `--silent`   | 結果を表示しない
`-s`, `--no-messages`         | エラーメッセージを表示しない

みどり（驚き）：
  ファイルの内容を自分の目で調べなくても良くなるね！
しげる（通常）：
  `grep` は見つかったら「成功」を
  見つからなかったら「失敗」を返すよ
  このことを利用して条件判定にも使えるんだ
  1回でもマッチすれば十分なので
  `-m1` オプションを指定すると
  処理が早く終わる場合があるよ
  また出力はいらないので
  `-s` と `-q` オプションが
  指定されることが多いんだ
みどり（笑顔）：
  面白そうなコマンドだね！

## 実践

しげる（通常）：
  `/etc/passwd` ファイルを検索してみよう
  ユーザ情報が保管されている設定ファイルなんだ
しげる（通常）：
  情報が「コロン(`:`)」区切りで格納されていて
  一番最初がユーザ名となっているよ

```bash
cat /etc/passwd
```

みどり（通常）：
  実際に見るのが理解しやすいよね！

```console
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/sbin/nologin
...
shigeru:x:1000:1000:Shigeru Wakagi:/home/shigeru:/bin/bash
midori:x:1001:1001:Midori Wakagi:/home/midori:/bin/bash
```

みどり（通常）：
  確かに「コロン(`:`)」区切りで色々な情報があって
  最初がユーザ名のようだね！
しげる（通常）：
  この中から `midori` ユーザの情報だけを
  表示したい場合にはどうすれば良いんだろう？
みどり（通常）：
  これではどうかな？

```bash
grep 'midori' /etc/passwd
```

しげる（通常）：
  確かに見つかるね
  だけど最初のユーザ名以外の場所に
  `midori` という文字列が入っていると
  その行も一緒に混じって表示されてしまうよ
しげる（通常）：
  ユーザ名が先頭に書かれていることを利用しよう！
みどり（通常）：
  それならこれでどうかな？

```bash
grep '^midori' /etc/passwd
```

しげる（通常）：
  さっきよりも良いね！
  でも仮に `midoriiro` ユーザがいたら
  このユーザの行も引っかかって表示されてしまうんだ
  こういうような場合を避けるにはどうしたら良いだろう？
しげる（通常）：
  ヒントは「情報はコロン(`:`)で区切られている」だよ
みどり（驚き）：
  あっ！
  それならこうだよね！

```bash
grep "^midori:" /etc/passwd
```

しげる（笑顔）：
  よくできました！
しげる（笑顔）：
  `grep` はこのように正規表現を指定して
  調べたい文字列をを探すことができるよ
