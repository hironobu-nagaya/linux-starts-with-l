set コマンド
お兄ちゃん！ シェルスクリプト途中のコマンドが失敗したら 後続のコマンドも同様に失敗しちゃたんだ まるで失敗のドミノ倒しだよ！ こういうのってなんとかならないかな？
もちろん対処できるよ！ 今回はシェルオプションについて学ぼう！
-e オプション
set はシェルにオプション変更を伝えるよ 
set -オプション
set +オプション
へぇ… シェルにもオプションがあったんだ！
しかも、動作している途中で オプションを切り替えられるんだね！
bash の -e オプションは コマンドが失敗したらスクリプト実行を そこで停止するオプションなんだ
ただし成否を評価に使っている部分の失敗は例外だよ if文やwhile文、AND 演算子 && や OR 演算子 || のような場合だね
まさに私が欲しかった機能だよ！ どうやって指定するの？
set に -e オプションを指定してあげれば良いよ シェルスクリプトでは冒頭に書くことが多いね 
set -e
失敗する可能性のあるコマンドを 一時的に許容してあげたい場合は どうすれば良いのかな？
オプションの - を + にすると そのオプションを解除する意味になるんだ そのようなコマンドの直前で set +e とすると良いよ
- で指定 + で解除… ちょっと混乱しそう
これらを組み合わせて このように書くと良いよ 
#!/bin/bash

# 通常は失敗したら停止
set -e

...

# ここだけ失敗を許したい
set +e
失敗の可能性があるコマンド
set -e

...
-v, -x オプション
せっかく set の話をしたので 他の便利なオプションについて紹介しよう！
えっ！ 興味ある！
内容が複雑になったとき処理を頭の中だけで追うのは大変だよね 以下のオプションを指定するとシェルスクリプト実行時にその内容が表示されるんだ
オプション 役割
-v 実行内容を表示
-x 実行内容の途中経過を表示
詳しく見たい所の始まりに set -vx 終わりに set +vx を指定してあげると 内容を理解しやすくなるよ 
#!/bin/bash

...

set -vx
詳しく見たい処理
set +vx

...
まとめ
今回は set でシェルのオプションを変更できることを学んだよ bash のオプションのうち -e, -v, -x オプションを覚えたよ！
僕がシェルスクリプトを書く時は冒頭に set -e を必ず書いているよ！
