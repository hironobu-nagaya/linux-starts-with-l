read コマンド
コマンドも 「対話的(interactive)」 「非対話的(non-interactive)」 という呼び方をするよ
へぇ！ どんな場合かな？
実行中にユーザへ 何も問い合わせないのを 「非対話的」と呼ぶよ 大多数のコマンドは 「非対話的」だね
確かにそうだね 普通はコマンドを実行したら すぐプロンプトが返るよね
一方、実行中にユーザの入力を 求めるようなものを 「対話的」と呼ぶんだ passwd のような コマンドだよ
そう考えると 「対話的」や 「非対話的」な コマンドがあるね
もしかして シェルスクリプトも 「対話的」にできるのかな？
もちろんできるよ！ 今回は「対話的」を実現する read を学ぼう！
使い方
read は入力を受け取り 変数に代入するよ 変数は複数指定できるけど 通常は1つと覚えておこう！ 
read [オプション]... [変数名]...
オプション役割
-s入力内容を隠す(パスワード入力時の挙動)
実践
挨拶をしてくれるgreet スクリプトを作ろう！まずはエディタでファイルを開いてね
vim greet
準備 OK だよ！
内容は以下のようにしてね
#!/bin/bash

echo -n "あなたの名前を教えて下さい: "
reod NAME
echo "こんにちは、$NAMEさん！"
シンプルだね！何をしてくれるのか想像がつくよ！
greet スクリプトに実行権限を付けてあげよう
chmod a+x greet
うん！さっそく実行してもいい？
もちろん！
./greet
えいっ！
あなたの名前を教えて下さい: ▌
私の名前は「みどり」です！
あなたの名前を教えて下さい: みどり▌
入力が終わったのでEnter だね！
あなたの名前を教えて下さい: みどり
こんにちは、みどりさん！
おおお！単純だけど温かみを感じるスクリプトだね！
read は入力を待ち受けるだけで「待ち受けメッセージ」は表示しないよスクリプトを実行した人はそのままでは入力待ちに気づかないので入力を待っていることを示すメッセージをecho で表示してあげると親切だよ
まとめ
今回は入力を受け取り変数に代入してくれるread を学んだよ！これを使えば「対話的」なスクリプトが書けるんだね！
黙って処理を続ける場合が多いけどユーザに聞くのが親切な場合もあるよそのようなときは read で入力を受け付けて処理しよう！
