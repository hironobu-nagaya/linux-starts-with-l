パス
お兄ちゃん！ パスワード変更とログアウトをしたけど 全然 Linux を操作している感じがしないよ！ もっと「操作している」感じのある コマンドってないのかな？
今回は今後の操作に必要になる用語を覚えよう！
これを覚えれば新しいコマンドを 教えてもらえるんだね？ 頑張って覚えるよ！
ファイル、ディレクトリ、パス
まずは「ファイル(file)」だよ
ファイル？ それくらいなら大丈夫だよ！
ファイルは特定の意味を持つデータを ひとつにまとめて扱えるようにしたものだね
でも… 何千何万ものファイルが1ヶ所にあったら 探すのも管理するのも大変だと思わない？
それを解決するのが… 確かフォルダでしょ？
そのとおり！ フォルダはファイルの入れ物のようなもので フォルダでファイルを整理できるね
Windows では「フォルダ(folder)」と呼ぶけど Linux や macOS では「ディレクトリ(directory)」と呼ぶんだ Linux を使う間は「ディレクトリ」と呼ぼう！
なるほど！ 用語は正しく使わないとね！
さらにディレクトリもファイルの一種なんだ なのでディレクトリの中に ディレクトリを入れられるよ
例えば a ディレクトリの中に b ディレクトリがあって さらにその中に c ファイルがある… って具合にね
こうした階層構造でファイルや ディレクトリを整理するよ
整理できるのはわかったけど… そのファイルの位置をどう表現するんだろう？
その位置を表すのが「パス(path)」なんだ パスは階層を「スラッシュ(/)」で区切って表現するよ
さっきの例だと「a/b/c」が 「c ファイルのパス」になるんだ
「ファイル」と「ディレクトリ」があって それらの位置を「パス」で表すんだね！
まとめ
今回は「ファイル」「ディレクトリ」「パス」 という用語を学んだよ！
次回はこれらを扱うコマンドを使っていくよ 実際にコマンドを打って使い方や感覚をつかもう！
