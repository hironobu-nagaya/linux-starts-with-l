補完
前回はシェルの便利機能「履歴」を学んだよ！
今回はもう一つの便利機能「補完(complementation)」を覚えよう！
使い方
ここでは cd /etc を入力しよう！
でも cd /e まで入力したらそこで手を止めてね
cd /e▌
うん！入力したよ！
この状態で「TAB キー」を押してみてね
cd /etc▌
わっ！自動で cd /etc って入力された！
ルートディレクトリ直下で/e で始まるのは /etc だけなんだだからシェルは「/etc を入力中」と判断して補完してくれたんだよ
候補が1つしか無い場合は自動で補完されるんだね！
候補が複数ある場合
今度は候補が複数ある場合を試してみよう！
常に候補が1つだけとは限らないからねそういう場合も教えてほしいよ！
ルートディレクトリ直下で /b で始まるのは/bin と /boot の2つがあるんだこれを例に手を動かしてみよう
まずは cd /b までを入力してみよう！
cd /b▌
うん！途中まで入れたよ！
この状態で「TABキー」を2回押してみてね
どうなるんだろう？
bin/  boot/
あっ！候補一覧が出てきたよ！
これで /b まで入力した時点の候補が/bin と /boot の2つがあるとわかるね
/b までは入力済みなので…/bin を指定したければ i を続けて入力してから補完/boot を指定したければ o を続けて入力してから補完
複数候補がある場合はこのように補完するんだ
これめっちゃ便利だね！
まとめ
補完は「TABキー」！便利機能はどんどん使っていくよ！
Linux に慣れている人は当たり前に使いこなしている機能だよ！快適な操作のためにもなるべく早く覚えよう！
