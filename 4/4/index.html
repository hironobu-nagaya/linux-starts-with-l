sudo とリダイレクト
お兄ちゃん！ sudo とリダイレクトを組み合わせると上手くいかないことがあるんだ 原因わかるかな？
それは sudo を使い始めの頃に多くの人がつまずくポイントだよ 今回は sudo と出力リダイレクトの仕組みとその解決方法を詳しく見てみよう
sudo とリダイレクトの問題
このコマンドを試したら失敗しちゃったんだ 
sudo cat > /etc/my-setting << EOF
USERNAME=midori
PASSWORD=irodim
EOF
実行結果はこうだったよ 
-bash: /etc/my-setting: Permission denied
このエラーの理由を説明するね sudo cat は cat を root ユーザ権限で実行しているんだ
うん！ そうだよね！
でもその後に続く部分 > /etc/my-setting は sudo の管理外だって気づいたかな？
えっ… そうなの？
実はそうなんだ リダイレクトはシェルが処理しているんだ
このシェルはみどりの権限で動いているから みどりが書き込めない場所へのリダイレクトは 「permission denied」になっちゃうというわけだね
へぇ… 勉強になるなぁ… でもこういう場合はどうすればいいの？
tee コマンド
tee は標準出力とファイルの二つへ出力を書き込むよ T字路の分岐みたいだからこの名前なんだ 
tee [オプション]... [ファイル]...
画面にもファイルにも出力… 欲張りさんなコマンドだね！
動作を確かめるために 以下のコマンドを実行してみよう！ 
echo "hello" | tee ~/tee-outout
まずは画面に hello が出るはずだよね？ 
hello
よし！ 画面出力は確認できたね！
もう一つの出力先も見てみよう 引数で指定した ~/tee-outout ファイルだね 
cat ~/tee-outout
こっちにも同じ内容が書かれていると思うよ！ 
hello
tee は出力を「ファイル」と「標準出力」の両方に送るんだ この tee に sudo をつけると書き込み権限が必要な場所にファイルを出力できるよ 
sudo tee /etc/my-setting << EOF
USERNAME=midori
PASSWORD=irodim
EOF
なるほど！ tee のファイル出力を出力リダイレクトの代わりに使うんだね
でも標準出力にも出力内容が表示されるね ちょっと気になるかも
確かにそう感じるよね でも画面に内容が表示されても害はないし簡単に対処できる方法なんだ どうしても気になるなら出力を /dev/null へリダイレクトしよう！
bash を使う
もう一つはシェル全体を管理者権限で実行する方法だよ
bash の -c オプションは指定された文字列をコマンドとして実行するよ 
bash -c 文字列
へぇ… でも指定する文字列がコマンドなら bach -c は無くても良いんじゃないかな？
普通ならそうだね でもリダイレクトを使うと事情が変わってくるんだ 
bash -c 'echo hello > /tmp/my-hello'
現行のシェルではなくコマンドで指定したシェルがリダイレクトをしてくれるんだね！
そういうことだよ！ その bash に sudo を付けると管理者権限でリダイレクトも含めた処理ができるんだ 
sudo bash -c 'echo hello > /etc/my-hello'
そっか！ 出力リダイレクトの処理も root ユーザ権限で行われるので書き込めるというわけなんだね！
みどりの場合はこのようにすれば動作するよ 
sudo bash -c 'cat > /etc/my-setting << EOF
USERNAME=midori
PASSWORD=irodim
EOF'
ちゃんとリダイレクトを使っていて理にかなっている書き方だね でもこの方法は理解して慣れるまで少し時間がかかるかも…
まとめ
今回は sudo と出力リダイレクトを組み合わせても 自分の権限の範囲しか書き込めないことを学んだよ この問題を回避するには tee や bash -c が使えるよ！
一見理解できない挙動もちゃんと原理に則って動いているんだ そういったものを紐解くことで Linux の理解は進んでいくよ！
